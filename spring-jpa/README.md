##### Section 6. 상품 도메인 개발
- Entity 자체에서 로직을 해결할 수 있는 경우, Entity 안에 비즈니스 로직을 작성
    - 객체지향적인 개발 (DDD)
    - 응집도가 높음
    - 데이터를 가지고 있는 쪽에 비즈니스 로직이 있는 것이 관리하기 좋음
- stockQuantity를 변경해야 할 경우
    - setStockQuantity() (setter) 으로 바깥에서 계산 X
    - 핵심 비즈니스 메서드를 가지고 변경
